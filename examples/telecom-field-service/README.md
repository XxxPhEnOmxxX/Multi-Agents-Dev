# Optional Domain-Specific Worked Examples

This directory contains optional, domain-specific worked examples that are intentionally kept out of the base template.

The base template stays domain- and platform-neutral. The examples here use a concrete domain (telecom/ISP field service: technicians, service orders, GPS check-ins) and concrete platform choices (Docker, VPS) to show what a fully contextualized session looks like in a real project.

Use these examples only if they match your project's domain. When adapting the template, prefer writing your own project-specific examples instead of copying these.

## Contents

- `architecture-session.md`: a question-driven architecture session for a field service management system.
