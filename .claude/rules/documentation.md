# Documentation Rules

## Responsabilidade

Use `technical-writer` para documentação técnica, documentação de API, base de conhecimento e release notes.

## Padrões esperados

- Documentação deve ser útil para executar, operar ou manter o projeto.
- Priorize exemplos reais e comandos verificáveis.
- Evite texto genérico sem aplicação prática.
- Se documentar API, inclua endpoint, método, payload, resposta e erros.
- Se documentar setup, inclua pré-requisitos, comandos e validação.

## Antes de finalizar

Verifique:

- se os comandos existem;
- se os caminhos citados existem;
- se a documentação está coerente com o código;
- se há informações sensíveis que não devem ser publicadas.
