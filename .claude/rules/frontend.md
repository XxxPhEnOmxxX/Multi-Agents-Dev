# Frontend Rules

## Responsabilidade

Use `frontend-specialist` para tarefas de interface, React, componentes, responsividade e acessibilidade.

## Padrões esperados

- Manter componentes pequenos e claros.
- Evitar estado global quando estado local resolver.
- Tratar carregamento, erro e sucesso.
- Validar layout em mobile, tablet e desktop.
- Preferir HTML semântico.
- Preservar o padrão visual existente do projeto.

## Antes de finalizar

Verifique:

- responsividade;
- acessibilidade básica;
- estados de carregamento e erro;
- consistência visual;
- build ou lint quando existir.
